# 编码说明

本项目所有文本文件统一使用 UTF-8 编码。

## 为什么要统一编码

项目中包含大量中文提示词、说明文案和安装脚本。如果文件被 Windows 默认代码页、GBK 或 ANSI 错误读取后再写回，就会出现类似中文被拆成奇怪汉字、emoji 变成异常字符、或整段内容变成连续问号的乱码。

## 修改文件时请注意

- Python 读写文本时统一使用 `encoding="utf-8"`。
- 编辑器保存编码请选择 `UTF-8`。
- 不要用未确认编码的命令批量重写中文文件。
- 打包前运行 `python health_check.py`，其中的“文本编码”检查会拦截典型乱码。

## 如果发现乱码

优先从最近的 `backups/` 目录恢复对应文件，或者根据 `config/` 中的 YAML 配置重新生成文本。不要直接在乱码基础上继续复制粘贴。
