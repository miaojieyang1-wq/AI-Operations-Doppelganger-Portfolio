# 设计决策记录

## 2026-05-16：运行时数据目录兜底

### 背景

在当前 Windows 环境中，默认用户数据目录 `LocalAppData/AI_Ops_Agent` 存在不可读取或不可写入的情况。原逻辑会在该目录上继续创建报告、索引、数据库和缓存，可能导致启动失败或环境检查卡住。

### 决策

保持“应用资源目录”和“用户可写数据目录”分离不变。`runtime_paths.py` 在未显式配置 `AI_OPS_DATA_DIR` 时，按顺序尝试以下目录：

1. 系统推荐用户数据目录
2. 用户主目录下的 `.AI_Ops_Agent`
3. 系统临时目录下的 `AI_Ops_Agent`

如果用户显式配置了 `AI_OPS_DATA_DIR`，则不自动降级，直接暴露配置错误，避免悄悄写到非预期位置。

### 影响

- 不改变业务逻辑、页面入口、Agent 流程或报告格式。
- 提升普通用户启动成功率。
- 临时目录兜底只在前两个目录不可用时启用，适合演示和容灾，不建议作为长期生产存储位置。
