# 变更记录

## 未发布

### 作品集资料更新

- 自然语言工作流新增本地快速规划：常见版本分析、竞品雷达、反馈清洗、问卷、访谈、活动、协作和诊断需求可直接生成 DAG，减少等待。
- 工作流规划 API 增加短输出限制和失败降级：外部接口不可用时自动回退到可执行默认流程。
- 静态工作流执行改为逐步刷新：每个步骤独立执行并更新卡片状态，执行中提示用户耐心等待。
- 步骤卡片增加完成摘要，用户可在完整报告生成前感知每一步的实际产出。
- 步骤间依赖上下文增加可配置压缩上限，避免多步骤链路反复传递超长报告导致后续调用变慢。
- 增加输入变更保护：生成工作流后如果用户修改原始需求，会提示重新生成并暂停执行旧流程。
- 冒烟测试补充自然语言工作流快速规划、规划失败降级和依赖上下文压缩用例。
- 新增“知识治理控制台”作品集文档，并加入作品集卡片与“了解我”回答口径，说明其作为独立 Streamlit 知识治理后台与 AI 运营分身 Agent 的低耦合同步关系。
- 优化生成中状态条，在原有旋转动效基础上加入“识别意图 → 检索素材 → 组织输出”的三步流转提示，降低等待时的开发感。
- 为“知识治理控制台”和“星铁战术引擎”作品集卡片增加 GitHub 仓库入口，使用者可在作品集页面直接点击查看独立项目。
- 将知识治理控制台和星铁战术引擎 GitHub 仓库内容同步到本地 `works/` 项目文件夹，并按公开仓库 README / 架构文档更新作品说明。
- 为十个协作运营角色增加 YAML 确定性规则与感知过滤器；协作讨论会按角色过滤不相关输入，信息不足时回退为前置声明或有限信息视角建议。
- 新增自然语言编排工作流入口，用户可用一句运营目标生成已有功能模块调用链并执行，结果按步骤展示数据依赖与执行状态。
- 强化自然语言编排在决策看板中的主入口地位，将原路线图主区域改为当前指令流程、执行状态和修改记录展示区。
- 为自然语言编排补充步骤滑入、思考占位、执行高亮、完成勾选、废弃标记和重规划遮罩等前端动效，增强工作流协作感。

### 阶段一：零风险基础加固

- 保留并补强 `.gitignore`，默认排除密钥、数据库、报告、索引、缓存、构建产物和快捷方式。
- 修复 `.env.example` 的中文乱码，补齐当前项目使用的环境变量模板。
- 将源代码、配置和文档中的 CRLF 文件统一转换为 LF，符合 Codex 自定义指令的编码与换行规范。
- 新增 `check_env.py`，用于手动检查 Python 版本、运行目录、关键目录和 API/演示模式状态。
- 修复 `check_env.py` 未读取项目根目录 `.env` 的问题，避免用户已经配置 Key 但检查失败。
- 修复 `check_env.py` 在部分 Windows 用户数据目录权限异常时可能卡住的问题，改为快速读写权限探测，保证快速返回且避免误判。
- 修复运行时默认用户数据目录不可读写时应用可能启动失败的问题，`runtime_paths.py` 会在非显式配置场景下自动尝试用户主目录与系统临时目录两个备用目录。
- 同步更新 `check_env.py` 的目录检查逻辑，使环境检查结果与应用实际兜底行为一致。
- 修复批量格式转换过程中 `agent_graph.py` 被误清空的问题，已从本地 Git 对象库恢复并通过冒烟测试。

### 阶段二：配置与硬编码隔离

- 版本分析界面改为优先从配置中心读取 `modules/version_analysis` 提示词，避免 UI 内嵌提示词与 YAML 配置长期分叉。
- 清理版本分析与竞品雷达生成流程中的无用临时变量，减少后续维护误判。
- 竞品雷达提示词加载封装为 `get_competitor_prompt`，继续保持配置中心优先、代码兜底的读取路径。
- 外部模型接口的连接超时、读取超时和重试次数加入 `config/app.yaml` 与默认配置，避免 API 调用长时间卡住。

### 阶段四：代码结构防熵增

- 拆分 `render_version_decision`，将版本分析、竞品雷达、文件选择、竞品分类、生成流程拆成独立小函数。
- `render_version_decision` 从约 150 行降至 31 行，主函数仅保留标题展示与子功能路由。
- 拆分 `render_user_insight`，将访谈助手、问卷工坊、反馈清洗、顶部说明和 Tab 选择逻辑拆成独立小函数，降低用户洞察模式的状态耦合。
- 用户洞察模式新增的小函数均控制在 30 行以内，预设问题写入继续使用临时值接管方式，避免再次触发 Streamlit 同名状态异常。
- 拆分侧边栏 API 连接面板，将状态初始化、输入渲染、连接校验、配置清除和状态展示拆成独立小函数，保留原有校验与脱敏展示行为。
- 拆分默认首页 `render_decision_dashboard`，将五阶段数据、流程总览条、阶段卡片、展开详情和连接器拆为独立结构与小函数，主函数降至 15 行。

### 阶段三：日志与错误处理

- 新增 `middleware.py`，提供统一日志格式和 `request_id` 注入工具，供后续渐进式接入。
- 优化 `api.py` 的接口处理流程，抽取统一的“生成结果 → 显式保存 → 返回保存状态”函数，减少重复逻辑。
- 为 API 报告持久化失败增加日志记录，保存失败仍不影响接口正常返回，但后续可排查具体失败原因。
- 为 `core.py` 的 Agent、RAG、协作讨论角色和协调者调用增加异常日志，保留原有兜底返回行为。
- 将 `rag_engine.py` 中运行时提示输出改为日志记录，避免后台检索降级时无排查线索。
- `api_runtime.py` 创建 OpenAI 兼容客户端时统一设置超时和最大重试次数，符合外部调用边界要求。

### 阶段六：可观测性

- `middleware.py` 提供 `RequestIdFilter` 与 `with_request_id` 装饰器，可为函数调用链路自动生成 `request_id`。

### 阶段七：交付与文档

- 新增 `smoke_test.py`，覆盖本地演示回答、空输入兜底、完整公式组直接读取三个核心冒烟场景。
- 更新 `README.md` 的自检说明，补充 `check_env.py`、`smoke_test.py` 与 `health_check.py` 的推荐执行顺序。
- 修复 `health_check.py` 语法检查会写入 `__pycache__` 的问题，改为内存编译，避免 Windows 权限异常导致误报。
- 管理后台登录区增加“演示模式：直接进入管理后台”入口，并明确说明这是作品集演示用模拟安全策略，真实部署应关闭。
- 修复演示模式下访谈助手被“调研目标”关键词误分类为问卷工坊的问题，访谈类任务现在优先匹配访谈演示结果。
- 修复访谈助手和问卷工坊预设示例按钮直接修改文本框同名 `session_state` 可能触发 Streamlit 状态异常的问题，改为安全的临时值注入方式。
- 修复从决策看板跳转到用户洞察或版本决策指定 Tab 时，直接修改 radio 同名 `session_state` 可能触发 Streamlit 状态异常的问题，改为通过默认索引安全切换。
- 修复访谈助手和问卷工坊预设按钮写入后文本框未正确接管新内容的问题，文本框现在由 `session_state` 统一管理，预设后仍可继续手动输入。
- 修复 LangGraph 任务调用前未同步侧边栏 API 配置的问题，避免问卷、访谈等工具继续使用 `.env` 中的旧 Key。
- 用户洞察生成失败时会识别 API Key 无效或缺失，并提示重新连接 API 或开启演示模式，不再只显示通用报错。
- 侧边栏 API 状态不再把 `.env` 中存在的 Key 直接显示为“已连接”，只有本次会话真实验证成功后才显示已连接；401 失败会自动重置连接状态。
- 自我诊断提示词和演示结果中移除“能力盲区”模块，诊断报告不再主动输出该字段。

### 阶段十五：战术引擎公式追问修复

- 修复关于我对话中先询问战术引擎数学思路、再仅输入“公式”时不会输出完整公式组的问题。

- 将 Streamlit 默认主按钮红色统一调整为项目主色青绿色，降低默认开发框架感，并保留禁用态和悬停态反馈。
- 增加 `prefers-reduced-motion` 兜底，尊重系统减少动态效果设置。

- `agent_graph.py` 已恢复到最近一次本地提交中的完整版本，避免 LangGraph 入口缺失。
